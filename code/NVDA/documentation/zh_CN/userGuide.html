1. 前言

1.1. 特色

透过语音朗读与点字， NVDA(Nonvisual Desktop Access) 可提供视障者操作微软窗口操作系统及其它应用程序。

主要特色包括：

  • 不需安装，可直接利用U盘或其它便携式保存媒体来操作
  • 提供有语音提示的简易安装版
  • 支持 Mozilla Firefox 或者微软 IE 浏览网页
  • 支持 Mozilla Thunderbird、Outlook Express、Windows Mail 或者 Windows Live Mail 收发电子邮件
  • 支持微软 Word、Excel
  • 支持可及性的 Java 应用程序
  • 支持 Adobe Reader
  • 支持 IBM Lotus Symphony
  • 支持命令提示符窗口
  • 可自动朗读鼠标指向的文本，及以声音提示鼠标所在位置
  • 支持许多型号的点字显示器

1.2. 国际化

让世界上所有的人，不管使用何种语言，都可以无障碍的公平使用科技是非常重要的，NVDA 当前除了英文版外，已被翻译成超过20种语言，包括：阿拉伯语、巴西语、克罗地亚语、捷克语、芬兰语、法语、加利西亚语、德语、匈牙利语、义大利语、日语、葡匋牙语、俄语、斯洛伐克语、西班牙语、泰语、繁体中文、简体中文、乌克兰语、南非语、波兰语及越南语等。

1.3. 可支持的语音合成器

除了提供前述多国文本消息与用户接口外，只要能取得到并安装在电脑上，透过语音合成接口程序的选择，NVDA 也提供用户采用不同的语言及不同的语音合成器，以便可读取文档或对象的内容。

默认提供一个多语系的语音合成器 eSpeak，此为一自由开放源代码的软件。

有关其它 NVDA 有支持的语音合成器的信息请参考 支持的语音合成器 章节。

1.4. 可支持的点显器

对于拥有点字显示器的用户，NVDA 可将信息以点字形式输出到点显器上。 关于支持的点字显示器信息，请参考支持点字显示器的章节。

NVDA 提供多语系，通常含括一级与二级点字码、以及电脑点字。

3. NVDA 程序安装

3.1. 安装版

若您的电脑上已下载了安装版的 NVDA，只要点击运行它，即开始进行安装。

当安装程序加载时，会伴随着音乐声。 加载后，会生成一个临时的提示消息窗口，以逐步引导安装。 注意：如果此时有旧版的 NVDA 正在运行，则会跳出一消息窗口，提示需将其关闭，按 [确定] 后，即将正运行的 NVDA 关闭，以便打开安装新的程序。 此步骤是必要的，如此一来，安装程序才可正确地更新。 如果电脑中已有安装版的 NVDA，则此时会将其解除安装，然后才开始进行安装新版的程序。

一旦成功的完成安装步骤，安装程序会提醒您“安装已完成”的消息。 如果是属于重新安装，则可能会跳出请求“重新开机”的消息，依其提示重新开机是非常重要的。 若没有依请求运行重新开机，则极有可能会发生难以预料的问题。 如果它并没有出现要重新开机的消息，则按 [完成] ，接着即自动打开新安装的 NVDA。

3.2. 便携版本

如果您已取得便携版的 NVDA，则只需将其单击解压缩，此时会跳出询问将此文件夹置放于何处的对话框，选定并确定后即完成解压缩。

4. 开始使用 NVDA

4.1. 启动 NVDA

如果您已安装了 NVDA 安装版，最简单打开的方式是利用按快捷键 Ctrl+ Alt+N，或从 [开始菜单] 中的 [程序集] 找到 NVDA 。 另一种启动它的方式是在 [开始菜单] 中在 [运行] 的对话框中输入 NVDA，然后再按 Enter即可。

至于若是使用便携版，则只要找到 NVDA 文件夹所存放的位置，将其打开后，利用按英文字母 N，找到 NVDA.exe，再按 Enter即可启动它。

当 NVDA 被启动时，您会听到一串上扬的声音，表示正在加载程序，稍等片刻，听到“NVDA 加载中”，即表示NVDA 已启动了。 等待的时间长短与电脑本身的等级有关，愈高等级需等待时间就愈短，而若是使用启动随身碟或 CD 内的NVDA 程序，则需要等待较长时间。 若等待时间稍长，NVDA 会提示“请稍候”。

如果没听到以上语音提示，而听到的是窗口错误的回应声，或是一串下沉的声音，那就代表 NVDA 加载没有成功，您需要重新查看可能引发的原因或向 NVDA 开发者回报此状况。 请连上 NVDA 官网查看如何回报程序错误的方式。

当首次启动 NVDA会跳出一个欢迎的对话框，其内包含了如何设置 NVDA 键及其菜单的基本消息，请参见以下章节以进一步了解其操作方式。 在此对话框还包括二个复选框，第一个是，是否要将 CapsLock 键修改成 NVDA 键，第二个是，是否每当 NVDA 启动时显示此对话框。

4.2. 有关 NVDA 快捷键

4.2.1. NVDA 组合键

除了在编辑区浏览文档仅需按数字键盘上的单一按键外，通常大多数的NVDA 相关快捷键均需同时按一个名为 NVDA 组合键的特殊键，外加另一个按键。

NVDA 可以被设置使用数字键盘 Insert、主键盘上的 Insert、或者 CapsLock 键，作为 NVDA 组合键。

默认状况是将前述的主键盘上的 Insert及数字键盘上的0设为 NVDA 组合键，简称为 NVDA 键。

假如你想使用 NVDA 键原有的功能(举例来说，如果你已设置 CapsLock 键作为 NVDA 键，但又要用它来转换大写锁定状态)，你可以快速连按此键两次。

4.2.2. 输入帮助

多数的 NVDA 组合键将于本使用手册中陆续被提出，但最简单探索其各种功能的方式是启动输入帮助的功能。

启动输入帮助的方式是按 NVDA+ 大键盘的1。 若要将此功能关闭则再按一次即可，此为一切换组合键的功能。 当此输入帮助被启动后，接下来的各单一按键或组合键并不会运行其功能，而是仅读出其所代表的功能而已。 因此，您可以试着按任何想按的键，以了解其功能。

4.2.3. 键盘配置

NVDA 快捷键当前有两种键盘配置，分别用在台式电脑与笔记本电脑上。 默认配置使用台式电脑，不过你可以从 NVDA 菜单的选项中的键盘设置切换到笔记本电脑配置。

台式电脑配置大量使用数字键盘（使用时要关掉 NumLock ）。 虽然大部分的笔记本电脑没有实体的数字键盘，但有一些可以利用 FN 键搭配在键盘右边的英文与数字（如7 8 9 u i o j k l 等等）来模拟数字键盘。 假如你的笔记本电脑没有此功能，或者无法关掉NumLock 键，你可能就会想要切换到笔记本电脑配置。

4.3. NVDA 菜单

NVDA 菜单提供选项、工具、保存或撤消设置、捐赠、退出等功能选项。

当 NVDA 已加载后，在任何窗口下，要打开菜单是按 NVDA+ N，另一种打开的方式是直触点击在通知区域内的 NVDA 图标按钮后，接着按鼠标右键即可。 若纯以键盘方式操作，则应先按 WINDOWS+ B，将焦点移到通知区域后，再以方向键找到 NVDA 图标按钮，接着按应用程序键（或称属性键），其位置在紧邻右 Ctrl 键的左边一个按键。 一但 NVDA 菜单打开后，可利用方向键移到所要选择的项目，接着按 Enter 以打开。

5. NVDA 导航

5.1. 系统焦点导航

利用 NVDA 在窗口操作系统上进行语音导航，最常用到的操作方式是使用标准的窗口键盘指令来移动，例如按 Tab 或 Shift+Tab 在各控件间前进或后退移动、利用按 Alt 打开菜单及以方向键在菜单内的项目间移动、以及使用Alt+ Tab 在打开的不同窗口间切换。 这些都会移动系统焦点到接收键盘指令的对象上。 当您这样操作时，NVDA 通常会读出焦点所在的对象相关的信息，包括：名称、格式、数值、状态、描述、快捷键及位置等。

以下是几个与获取焦点所在对象相关信息的指令：

名称 快捷键 描述
读出当前焦点 NVDA+tab 读出当前焦点所在的对象或控件，连按两次将拼读其信息。
读出窗口标题 NVDA+t 读出当前作用中的窗口标题，若连按两下 T 将标题拼读出来，连按三下T 则将标题复制到剪贴板。
读出作用中的窗口内容 NVDA+b 读出当前作用中窗口所有的控件(尤其在对话框中很有用)。

5.2. 系统编辑光标导航

当系统焦点是在一个可编辑的对象内，则焦点改以一个可编辑的插入点形式显示，我们称此为编辑光标所在（或简称光标）。 您可在此处进行编辑操作，包括输入字符、删除字符、或选择字符串等。 您可以利用按上下左右方向键来移动此编辑光标，对于每个操控，除了系统会回应应有的动作外，NVDA 会另外将其读出来。 这些操控包括：移动一个字符、一个字词（在英文是指一个单字）、一行字、一句话、或甚至一个段落。 另外，选择或不选择文本也都能朗读出来。

5.3. 对象导航

如果你仅想在当前应用程序或操作系统的焦点所在附近作些小幅度的导航，可利用 NVDA 提供的对象导航功能来达成。 这项功能使你可以在对象中移动，并取得对象的信息，例如文本、按钮、复选框、滑块、列表与可编辑的文本区块等。 当你移动到某一对象时，NVDA 会使用类似于朗读系统焦点的方式来朗读。

因为对象是被以层级架构的方似组合起来，所以并无法在对象中随意移来移去。 你必须先进入到某些对象内，才可以访问到其内的对象。 举例来说，当遇到列表时，无法立即直接在每个项目中移动，须先导航进入列表内，才能访问其内的其它项目。 你可以离开这个列表跳到下一个列表内再以同样的方式读取其内的项目。 同样地，当遇到工具栏时，你必须先导航进入工具栏对象中的第一个子对象，才可再操作其它的控件。

默认情况下，对象导航会跟随系统焦点移动，这个功能可以被激活或停止。

请注意：要让对象的浏览光标正常运作，NumLock 键应在关闭状态。

6. 虚拟缓冲

对于那些复杂而通常仅供阅读的文档如网页的页面，NVDA 是将其转换使其显示在一个虚拟缓冲区内。

虚拟缓冲是指将网页的内容以平面方式显示，而您可用方向键去读它，网页上的信息，不管是否为文本，如链接、标题等都可以随着光标的移动而朗读出来。

有二种操作模式来与虚拟缓冲区内的内容互动，包括：

·?浏览模式：允许使用方向键四处移动读取文本数据。

·?焦点模式：允许与那些控件如编辑区、组合框及单选按钮等互动，其操作方式如一般您在网页上的操作方式。

当在浏览模式时，若 Tab 到或鼠标单击到一个控件，则会自动切换到焦点模式。 请注意，如果在一个控件，按空格键或Enter也均可切换到焦点模式。

假如 NVDA 被自动地切换到焦点模式，移动到对象或者单击到对象，而不需要焦点模式时，将会自动地切换回浏览模式。 按下 Esc 键也能切换浏览模式。 你也可以手动切换焦点模式与浏览模式。 但是当手动切换到焦点模式时，NVDA 将被锁住在这个特定模式，不会再被自动切换，除非你在手动切换成浏览模式。

此为一复选框，若选择则 NVDA 会读出在屏幕上出现的工具提示消息。 许多窗口操作系统上的应用程序或控件会提供一个小的工具提示消息，当光标移到该对象附近时，即会朗读出来。

读出气泡提示

若选择此控件则当气泡提示窗口出现时会朗读出来。 气泡提示窗口与工具提示消息窗口类似，但通常其尺寸略大，提供与系统事件相关的消息，如区域网络在线或与系统安全相关的警告消息。

读出对象的快捷键

若选择此控件，NVDA 会将对象或控件及其连带的快捷键一起读出，例如在菜单列上的 [文件] 菜单，其快捷键是 Alt+F。

读出对象位置信息

若选择此控件，则NVDA 会读出焦点所在对象的位置，例如第1项，共4项。

读出对象描述

若不需读出焦点所在对象的描述，则不需选择此控件。

读出进度蓝
台式电脑快捷键 NVDA+u
笔记本电脑快捷键 Ctrl+NVDA+F2

这个选项控制 NVDA 如何朗读出进度蓝的最新情况。

包括以下各选项：

  • 关闭：不读出任何信息。
  • 读出进度百分比：每当进度蓝有百分之十的改变时即朗读出来。
  • 发出哔哔声：随进度完成的状况而逐渐提高音调。
  • 读出进度百分比及发出哔哔声：每当进度蓝有百分之十的改变时即朗读出来及发出哔哔声。
读出背景进度蓝

若选择此控件，尽管此进度蓝并不是在作用中的前景窗口也会读出其进度变化。 在设置此情况下，如果将显示有进度蓝的窗口最小化或切换到另一个窗口以便可继续作别的任务，则NVDA 仍会持续监控这个进度蓝，以便让用户掌握该项任务的进度。

使用屏幕实际配置(当有支持时)

读出排版表格

此为一复选框的控件，若没有选择则NVDA 仅朗读那些数据表格。 此为一般常见的表格。 若选择此控件，则浏览光标遇到网页上的排版表格也会读出来。

读出链接、标题之类的元素

此为一复选框。 以前的NVDA 版本，当浏览光标移到一些网页元素时，可设置是否提供将其读出来，这些网页上的元素包括：链接、标题、表格及列表等。 新版本的设置请参见 文档格式设置。 这一节的适用对象不仅限于网页上的虚拟缓冲区上的元素，其它尚有微软的 WORD 及 PDF等。

自动启动焦点模式

若选择此控件，当焦点移到一个复选框，则会自动切换至焦点模式，例如利用按 Tab 移至一个复选框的控件。

当光标移至编辑区自动启动编辑模式

若选择此控件，当利用方向键移至一个可编辑的文本框时，会自动切换至编辑模式，可在编辑光标处输入字符或删除字符。 若以方向键移出此文本框，则又会自动切换成浏览模式。

以音效提示焦点模式与浏览模式的切换

若选择此控件，则当在浏览网页时，若有任何焦点模式与浏览模式切换，NVDA 将会发出特殊的音效，而不是以朗读的方式提示。

7.1.11. 文档格式设置 (Ctrl+NVDA+d)

大部分在此对话框中的复选框控件，都是用来设置要如何让NVDA 朗读微软 WORD 或 wordpad的文档格式相关的信息，例如：如果选择 [读出字体名称] 这个控件，当光标移到有字体名称改变的地方，就会自动朗读出来。

可以设置朗读的项目包含：

  • 字体名称
  • 字体大小
  • 字体属性
  • 型态
  • 对齐方式
  • 文本样式
  • 表格
  • 表格的行和列标题
  • 页码
  • 行数
  • 拼错字
  • 链接
  • 标题
  • 列表
  • 引述区
  • 书签
朗读光标经过后的格式变换

假如启动这项设置，　NVDA 将设法检测光标所经过的字符的格式变换，并将其朗读出来，这样设置，将会降低 NVDA 的效率。

NVDA 会检测到编辑光标／浏览光标所在位置的格式，在某些情况下如果不影响 NVDA 的效率，将会读出整行格式的变化。 当文档格式很重要时，例如使用微软 Word 来校订文档时，可激活这个功能。

7.1.12. 读音字库

针对某些字符串需要改变其读音方式，此对话框提供了修改的设置，包括有三个字库：

  • 默认字库：在这个字库修改会对所有的 NVDA 发生作用，含不同语音引擎及不同的应用程序。
  • 音字库：仅对当前所使用的语音引擎有作用。
  • 暂存字库：仅对当前使用的 NVDA 发生效用，若关掉 NVDA再重新打开后该字库即告失效。

所有字库的对话框都含有读音处理的槼则列表。 这个对话框也包含添加、编辑与删除的按钮。

若要添加改变一个字符串的读音，则按 [添加] 这个按钮，接着依序在可编辑框内填入原来字符串及改变读音的字符串等，接着按一下 [OK] 按钮，则会发现读音字库的列表已将此项添加了。 最后若要离开此对话框，要记得再按一下 [OK] 按钮，才算完成添加的操作程序。

读音字库采行的槼则是允许你改变一个字或字符串的读音，举最简单的例子是，如果你想让一个英文本 bird 改变读成 frog。 其作法是在打开的 [添加] 对话框中，于 [原来文本] 的字段填入 bird，而在 [代替文本] 字段填入 frog，您也可在 [注解] 字段加注“将 bird 改成 frog的读音”，也可以留空白。

NVDA提供的这个功能相当强，您也可以设置 [大小写需相符] ，这是由一个复选框来控制，默认值是忽略大小写，也就是未选择。

一个 [正则表达式] 是指一个包含特殊符号的“样式”，它可用以一次相应到数个字符或一组数字， [正则表达式] 不在本使用手册的介绍范围，但您可上网找到许多入门教学的信息。 （中文版附注：修改读音字库的功能在一些简写或缩写的英文本尤其好用，例如原来的读法不易理解，就可采用此功能将其改变成个人可理解的读音。 由于英文本与字之间是以一个空格区分，在技术上比较容易正确显示，而中文本在字与字之间并没有空一格，则较不易采行。 ）

7.2. 保存与撤消设置

默认情况下，NVDA 并不会自动保存您所作的一些设置更改，所以如果没有保存它，当再重新打开 NVDA 时，这些修订即告消失，回到原先的设置。 不过 NVDA 另有提供修改此默认状态的功能，您可以进到偏好设置内的一般设置去修改该复选框控件。

如果您的 NVDA 应用程序是安装在一个只读的保存媒体，如一片光碟上，则您将无法保存设置的更改，若你作此动作，系统会给一个警告消息。

如果您不小心改变了设置，想要回到原先的设置，则可打开 NVDA 功能菜单中运行 [撤消设置] 这个项目。

另一种好用的作法是使用下列的 NVDA 快捷键：

名称 台式电脑快捷键 笔记本电脑快捷键 描述
保存设置 Ctrl+NVDA+c Ctrl+NVDA+c 保存当前的设置，当离开 NVDA 时，设置的数据不会丢失。
撤消设置 Ctrl+NVDA+r Ctrl+NVDA+r 将 NVDA 回滚到最后一次保存的设置。

7.3. 配置文件与用户数据

NVDA 可携式版将所有的设置、个性化应用模块与个性化驱动程序都保存在 NVDA 目录下的一个称为用户配置文件（userConfig）的目录中。

安装版则将其保存在窗口用户数据（user profile）中的一个特定的　NVDA 目录， 这表示在同一台电脑中每一个用户可以拥有自己的 NVDA 设置。 在安装版中，你可以从 [开始] 菜单进到 [所有程序] 中，找到用户配置文件目录。

关于登录界面或者 UAC 界面中的 NVDA 使用设置，保存在 NVDA 安装目录中的系统配置文件（systemConfig） 目录下。 通常这个配置文件不应该被编辑。 若要更改登录界面／UAC 界面的设置，必须要先设置好从 Windows 登录界面的 NVDA 设置，然后使用在一般设置对话框下的按钮，下指令给 NVDA，将设置保存提供给登录界面使用 。

9.1. eSpeak

9.3. Microsoft Speech API 第五版 (SAPI 5)

SAPI 5 是微软语音合成软件的标准版本。 遵循这个标准的语音合成器可以从许多公司或网站中购买或者免费下载。 你的系统中可能已预先安装至少一个 SAPI 5 的声音在里面。 当在 NVDA 使用这个合成器时，所有在你的系统中已安装的 SAPI 5.0 引擎提供的声音都可被使用（可以从NVDA菜单里的 语音设置 选用 或利用 快捷键设置语音）。

9.4. Audiologic Tts3

这是一个专门为义大利语的商务用语音合成器。 你必须要安装它，才可以在 NVDA 上使用。 若要了解更详细的信息，请浏览 Audiologic 网站 www.audiologic.it 。

9.5. Newfon

11.1. 重新映射按键任务和其他快捷键

用户可以重新映射或者覆盖nvda的快捷键(例如按键)提供到一个专门的脚本，此脚本位于用户的NVDA配置表中。 文件名为 gestures.ini。

此文件使用标准ini句法。 文件可能包含多个部分，每个部分可能有一个或多个小节。

每个小节提供映射给脚本，使用NVDA中特定的Python模块和类。

  • 小节名称必须采用Python模块和类，并以点(.)分隔。
  • 每个项的检索是设定该快捷键的脚本名。 或者，你也可以使用“None”将原先设定的快捷键从对应的脚本中解除。
  • 项值是一个以逗号分隔的快捷键标识符表单，以对应不同的快捷键。

快捷键标识符由这些部分组成：一个双字母设备码（kb代表键盘,bd代表点显器）,一个写在括弧中的可供选择的辅助设备或键盘设计；一个冒号，以及由加号分隔的一个或多个按键名称。

为了查看快捷键标识符，脚本名称以及所包含的芬类和模块，你可以：

  1. 打开输入帮助。
  2. 通过按键(按快捷键)。
  3. 关闭快捷键帮助。
  4. 点击NVDA工具菜单中的记录浏览器。 以上的一个或多个步骤均可提供你所发出的有关快捷键的信息，包括module.class和可能绑定的脚本。

下面是一个如何将NVDA+shift+t添加到日期脚本的例子。

为了找到正确的日期脚本名称和模块.分类，你需要打开快捷键帮助，按下NVDA+F12（这是通常的进入日期脚本的方式）。 然后关闭快捷键帮助，查看记录浏览器。

在末尾可以看到： 

  INFO - inputCore.InputManager._handleInputHelp (13:17:22):
  Input help: gesture kb(desktop):nvda+f12, bound to script dateTime on globalCommands.GlobalCommands

从这里可以看到，脚本名是dateTime模块.类是 globalCommands.GlobalCommands。

如果此文件尚未生成，你需要在用户配置表中建立一个名为gestures.ini的文本文档，并添加下列内容： 

  [globalCommands.GlobalCommands]
  	dateTime = kb:NVDA+shift+t

此内容将把任意键盘上的快捷键NVDA+shift+t设定在日期脚本当中。

注意，原来设定的NVDA+f12仍然有效。 如果你想解除这个设定，你需要添加以下内容： 

  	None = kb:NVDA+f12

虽然你可以随意将脚本和任何可利用的按键绑定，但是使用键盘上的alt键还是存在问题。 NVDA还提供组合键如 shift,control,alt键给操作系统，即使这些组合键都是设定在一个脚本中。这样的话，如果你在一个快捷键中使用alt，按下此键的组合将可能在激活菜单栏的同时也将执行这个脚本。 因此，最好还是只使用shift,control和NVDA作为组合键。
