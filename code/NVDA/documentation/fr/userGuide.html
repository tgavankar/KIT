1. Introduction

NonVisual Desktop Access (NVDA) est une revue d'écran gratuite et open-source pour le système d'exploitation Microsoft Windows. En donnant des informations via une voix synthétique et le Braille, il permet aux personnes aveugles ou malvoyantes d'accéder à un ordinateur sans coût additionnel par rapport à une personne voyante. NVDA est développé par NV Access, avec des contributions de la communauté.

1.1. Caractéristiques générales

En restituant l'information par synthèse vocale ou en Braille, NVDA permet aux personnes aveugles et mal-voyantes d'utiliser le système d'exploitation Windows ainsi que beaucoup d'applications disponibles dans cet environnement.

Ses points forts sont :

  • La prise en charge d'applications grand-public telles que des navigateurs Web, clients e-mail, programmes de tchat sur Internet et suites bureautiques
  • Un synthétiseur intégré prenant en charge plus de 43 langues
  • l'annonce, quand c'est possible, d'informations de formatage du texte telles que le nom et la taille de la police, le style et les fautes d'orthographe
  • L'annonce automatique du texte sous la souris et, en option, l'indication sonore de la position de la souris
  • La prise en charge de nombreux afficheurs braille
  • La possibilité de s'exécuter entièrement depuis une clé USB ou tout autre média portable sans avoir à installer
  • Un installateur parlant facile à utiliser
  • Une traduction dans beaucoup de langues
  • La prise en charge des environnements Windows modernes, incluant les versions 32 et 64 bits
  • L'accès à l'écran de connexion à Windows et aux autres écrans sécurisés
  • Prise en charge des interfaces d'accessibilités communes telles que Microsoft Active Accessibility, Java Access Bridge, IAccessible2 et UI Automation (UI Automation n'est supporté qu'à partir de Windows 7)
  • Support de l'invite de commandes Windows et des applications en mode console.

1.2. Internationalisation

Il est important que chacun, partout dans le monde, quelle que soit sa langue, ait le même accès aux technologies. A ce jour, NVDA a été traduit en 36 langues : l'Afrikaans, l'Albanais, l'Allemand, l'Arabe, le Bulgare, le Chinois Traditionnel, le Croate, le Danois, l'Espagnol, le Finnois, le Français, le Galicien, le Géorgien, l'Hébreu, le Hongrois, l'Hindi, l'Islandais, l'Italien, le Japonais, le Mandarin Taïwanais, le Néerlandais, le Norvégien, le Polonais, le Portugais, le Portugais Brésilien, le Roumain, le Russe, le Serbe, le Slovaque, le Suédois, le Tamoul, le Tchèque, le Thaï, le Turc, l'Ukrainien et le Vietnamien.

1.3. Support des synthèses vocales

En plus de proposer ses messages et son interface en plusieurs langues, NVDA permet également à l'utilisateur de lire les contenus dans n'importe quelle langue, dans la mesure où il possède un synthétiseur parlant cette langue.

NVDA est fourni avec eSpeak, un synthétiseur de parole gratuit, open-source, multilingue.

Pour des informations concernant les autres synthétiseurs de parole pris en charge par NVDA, reportez-vous à la section Synthétiseurs de paroles pris en charge.

1.4. Support du Braille

Pour les utilisateurs possédant un terminal braille, NVDA peut afficher ses informations en Braille. Reportez-vous à la section Terminaux braille pris en charge pour les informations concernant les terminaux braille.

NVDA propose de nombreux codes braille couvrant un grand ensemble de langues. Dans bien des cas, le Braille intégral et abrégé sont disponibles. Pour les utilisateurs francophones, le Braille français unifié est disponible en intégral et abrégé.

2. Configuration système requise

  • Systèmes d'exploitation : Toutes les variantes, 32 ou 64 bits, de Windows XP, Windows Vista et Windows 7 (y compris les versions Serveur).
  • Mémoire : 256 mo de RAM ou plus.
  • Vitesse processeur : 1.0 ghz ou plus.
  • Environ 50 MO d'espace disque.

3. Se procurer NVDA

Si vous n'avez pas encore une copie de NVDA, vous pouvez la télécharger sur nvda-project.org.

Rendez-vous à la section "Download" où vous trouverez un lien pour le téléchargement de la dernière version de NVDA.

NVDA se présente actuellement sous deux formes. L'une comporte un installateur qui vous guidera vocalement pour l'installation de sorte que vous pourrez ensuite lancer NVDA à volonté par le menu Démarrer. L'autre est une archive autoextractible qui contient une version complètement autonome de NVDA, qu'on peut lancer de n'importe où, y compris d'une clé USB ou d'un CD.

Pour accéder à l'écran de connexion à Windows, au contrôle d'accès utilisateur (UAC) ou d'autres écrans sécurisés ainsi qu'aux applications tournant en mode administrateur sous Vista ou Seven,, vous devez installer NVDA sur votre système en utilisant l'installateur. Ces écrans ne sont pas accessibles avec une version portable de NVDA.

4. Installer NVDA

4.1. L'installateur de NVDA

Si vous avez déjà l'installateur NVDA, appuyez simplement sur entrée ou double-cliquez sur le fichier et l'installateur va démarrer.

Pendant que l'installateur se charge, vous entendrez la musique d'installation de NVDA. Une fois chargé, une copie temporaire de NVDA vous permettra de suivre les dialogues d'installation et installera NVDA. Notez que si une autre copie de NVDA est active au moment de l'installation, celle-ci sera arrêtée. Cela est nécessaire pour que l'installateur puisse mettre à jour tous les fichiers modifiés. Si une version précédente de NVDA est déjà présente sur votre système, celle-ci sera désinstallée puis le programme démarrera l'installation de la nouvelle version.

Quand vous aurez fini avec succès l'installation, l'installateur vous indiquera que l'installation est terminée. S'il s'agit d'une réinstallation, il peut arriver qu'il vous demande de redémarrer le système. Il est très important d'effectuer ce redémarrage pour ne pas risquer de conséquences indésirables. Si l'installateur ne demande pas de redémarrage, un appui sur la touche Entrée arrêtera l'installateur et démarrera la nouvelle copie de NVDA.

4.2. NVDA portable

Si vous avez la version portable de NVDA, il vous suffit d'appuyer sur Entrée ou de doublecliquer sur le fichier. Le programme vous demandera le répertoire où vous désirez placer les fichiers sur votre système.

4.2.1. NVDA Portable sur une clé USB

Il est possible de mettre NVDA sur une clé USB de manière à pouvoir l'utiliser sur d'autres ordinateurs.

Quand la version portable demande où vous désirez la décompresser, entrez l'unité de votre clé USB plus, éventuellement, le nom d'un sous-répertoire où vous désirez placer NVDA.

Si, par exemple, votre clé USB est en E:\, vous pouvez entrer E:\nvda, ce qui placera NVDA dans le sous-répertoire nvda de votre clé USB.

Notez que si cela vous permet de disposer partout de NVDA et de ses paramètres, le simple fait de brancher la clé USB ne démarrera pas automatiquement NVDA. Vous devrez lancer manuellement NVDA depuis l'explorateur Windows où le dialogue "Exécuter".

Pour des information sur la manière de démarrer automatiquement NVDA sur une clé USB, veuillez consulter cet article : http://www.nvda-project.org/wiki/RunningAutomaticallyFromAUSBDrive

5. Prendre en main NVDA

5.1. Démarrer NVDA

Si NVDA a été installé à l'aide de l'installateur, alors il suffit, soit de frapper la combinaison control+alt+n, soit de choisir dans le menu Démarrer / Programmes / NVDA / NVDA. Vous pouvez aussi taper NVDA dans le dialogue Exécuter puis presser Entrer.

Pour démarrer la version portable, rendez-vous dans le dossier où NVDA a été décompressé, et appuyez sur entrée ou double-cliquez sur nvda.exe.

Au démarrage de NVDA, on entend d'abord une série de sons ascendants (indiquant que NVDA se charge). Suivant la rapidité de votre ordinateur, ou si vous exécutez NVDA à partir d'une clé USB ou d'un autre périphérique lent, le démarrage peut demander un certain temps. Si le démarrage est vraiment lent, vous entendrez le message "Chargement de NVDA, veuillez patienter...".

Si vous n'entendez rien de tout cela, ou si vous entendez le signal sonore d'erreur de Windows, ou une série de sons descendants, cela indique un problème de NVDA et vous pourrez, si vous le voulez, envoyer un rapport d'erreur au développeurs. Rapportez-vous au site Web de NVDA pour les détails de la procédure.

Quand NVDA démarre pour la première fois, vous êtes accueilli par une boîte de dialogue vous donnant des informations de base sur la touche NVDA et le menu NVDA. Veuillez consulter les sections ci-après pour plus d'information sur ces sujets. La boîte de dialogue contient également deux cases à cocher. La première vous permet de choisir si vous voulez utiliser la touche de verrouillage majuscules comme touche NVDA et la seconde permet d'indiquer si ce dialogue d'accueil doit apparaître à chaque démarrage.

5.2. À propos des commandes clavier de NVDA

5.2.1. La touche NVDA

La plupart des commandes spécifiques de NVDA se font en pressant la touche NVDA en conjonction avec une ou plusieurs autres touches. Les fonctions de revue de texte qui n'utilisent que le pavé numérique constituent une exception.

NVDA peut être configuré pour que les touches Insert du pavé numérique, Insert du clavier étendu ou verrouillage majuscules puissent être utilisés comme touche NVDA.

Par défaut, les touches insert du pavé numérique et du clavier étendu sont définies comme touche NVDA.

Si vous voulez utiliser l'une des touches NVDA dans sa fonction originale (par exemple si vous voulez utiliser verrouillage majuscules pour verrouiller les majuscules après l'avoir définie comme touche NVDA) vous devrez appuyer deux fois rapidement sur la touche.

5.2.2. Aide Clavier

Beaucoup de touches de commande sont mentionnées dans la suite de ce guide, mais une manière simple d'explorer les différentes touches de commande est d'activer l'aide clavier.

Pour activer l'aide clavier, appuyez sur NVDA+1. Pour la désactiver, appuyez à nouveau sur NVDA+1. Quand l'aide clavier est activée, l'appui sur une touche annonce sa fonction (si elle en a une). La fonction n'est pas réellement exécutée, vous pouvez donc presser n'importe quelle touche sans crainte.

5.2.3. Les configurations Clavier

NVDA comporte actuellement deux configurations clavier : une pour les ordinateurs de bureau (Desktop) et une pour les ordinateurs portables (Laptop). Par défaut, NVDA est configuré pour un ordinateur de bureau. Pour passer en configuration ordinateur portable, allez dans le menu Préférences > Paramètres du clavier.

La configuration pour ordinateur de bureau fait un usage intensif du pavé numérique (avec le verrouillage numérique désactivé). Bien que la plupart des ordinateurs portables ne comportent pas de pavé numérique, certains peuvent l'émuler en maintenant enfoncée la touche FN puis en pressant les lettres et les chiffres à la droite du clavier ( 7 8 9 u i o j k l etc). Si votre ordinateur ne vous permet pas de le faire, ou ne vous permet pas de passer en verrouillage numérique désactivé, vous pouvez alors utiliser la configuration pour ordinateur portable.

5.3. Le menu NVDA

Le menu NVDA vous permet de gérer les paramètres de NVDA, d'obtenir de l'aide, sauvegarder ou recharger la configuration, modifier le dictionnaire de prononciation, accéder à des outils additionnels ou quitter NVDA.

Pour accéder au menu NVDA depuis n'importe où dans Windows pendant que NVDA est en fonction, appuyez sur NVDA+n. Vous pouvez également y accéder par la barre des tâches système. Faites un click droit sur l'icône de NVDA située dans la barre des tâches système, ou accédez à la barre des tâches système en pressant Windows+b,,pressez flècheBas jusqu'à atteindre l'icône NVDA puis pressez la touche Application généralement située à côté de la touche Control de droite. Quand le menu apparaît, vous pouvez utiliser les flèches pour naviguer dans le menu, et la touche entrée pour activer un élément.

6. Naviguer avec NVDA

NVDA vous permet de naviguer dans le système ou de l'explorer de différentes manières, soit en interaction normale, soit en revue.

6.1. Objets

Chaque application et le système d'exploitation lui-même se compose d'un grand nombre d'objets. Un objet est un élément simple tel qu'un morceau de texte, un bouton, une case à cocher, un curseur, une liste ou une zone d'édition.

6.2. Naviguer avec le Focus système

Le focus système, ou plus simplement focus, est l'objet qui reçoit les touches tapées au clavier. Par exemple, si vous tapez du texte dans une zone d'édition, cette zone d'édition a le focus.

La manière la plus commune de naviguer dans Windows avec NVDA, est de simplement se déplacer avec des commandes clavier normales, comme Tabulation et Majuscule+Tabulation pour se déplacer entre les contrôles, Alt pour aller au menu puis Flèches pour naviguer dans les menus, utiliser Alt+Tabulation pour passer d'une application à l'autre. Cela déplace le focus système, c'est-à-dire l'objet qui reçoit les touches que vous tapez au clavier. Quand vous faites cela, NVDA donne des informations sur l'objet ayant le focus, telles que son nom, son type, son état, sa valeur, sa description, son raccourci clavier et sa position.

Quelques touches de commande utiles pour naviguer avec le focus :

Nom Touche Description
Annonce du focus courant NVDA+tab annonce l'objet en cours ou le contrôle qui a le focus système. Une double pression épelle l'information
Annonce du titre NVDA+t Annonce le titre de la fenêtre active. Une double pression épelle l'information, une triple pression la copie dans le presse-papier

Quand un objet qui autorise la navigation ou l'édition de texte a le focus, vous pouvez vous déplacer dans le texte en utilisant le curseur système ou curseur d'édition.

Quand le focus est sur un objet ayant un curseur d'édition, vous pouvez utiliser les flèches, les touches début, fin, page précédente, page suivante pour vous déplacer dans le texte. Vous pouvez également modifier le texte si le champ le permet. NVDA annonce le déplacement par caractère, mot ou ligne, et annonce aussi la sélection et la désélection de texte.

6.4. Naviguer par Objet

La plupart du temps, vous travaillerez avec les applications en utilisant des commandes qui déplacent le focus et le curseur. Cepandant, vous pouvez parfois avoir besoin d'explorer l'application en cours ou le système sans déplacer le focus. Vous pouvez aussi avoir besoin d'accéder à des objets inaccessibles par les commandes clavier habituelles. Dans ces cas, vous pouvez utiliser la navigation par objet.

La navigation par objet vous permet de vous déplacer parmi les différents (objets #Objects) et d'obtenir des informations à leur sujet. Quand vous atteignez un objet, NVDA l'annoncera de la même manière qu'il annonce le focus système. Si vous désirez explorer tout le texte tel qu'il se présente à l'écran, vous pouvez également utiliser la revue à plat.

Pour ne pas avoir à errer entre tous les objets présents dans le système, ceux-ci sont groupés hiérarchiquement. Cela signifie que vous devez entrer dans certains objets pour voir les objets qu'ils contiennent. Par exemple, une liste contient des éléments de liste, vous devez donc entrer dans la liste pour voir ces éléments. Si vous êtes sur un élément de liste, aller à l'élément suivant ou précédent vous amènera à un autre élément de la même liste. Si vous allez sur l'objet contenant les éléments de la liste, vous serez ramené à la liste. Vous pourrez alors la quitter si vous désirez accéder à d'autres objets. De la même manière, une barre d'outils contient des contrôles, vous devrez donc entrer dans la barre d'outils pour accéder aux contrôles qu'elle contient.

L'objet en cours de revue s'appelle l'objet navigateur. Quand vous atteignez un objet, vous pouvez l'examiner en utilisant les commandes de revue de texte. Par défaut, l'objet navigateur suit le focus système, mais ce comportement peut être désactivé.

Note : Par défaut, le Braille suit le focus et le curseur système plutôt que la navigation par objet ou la revue de texte. Si vous voulez suivre la navigation par objet ou la revue de texte, vous devez configurer le Braille pour qu'il soit attaché à la review.

6.5. Revue de Texte

NVDA vous permet de lire le contenu de l'objet navigateur courant par caractère, mot ou ligne. Ceci est particulièrement utile dans la console de commandes Windows et d'autres applications où le curseur système est inexistant. Par exemple, cela peut servir à consulter le texte d'un long message dans un dialogue.

Quand vous déplacez le curseur de revue, le curseur système ne bouge pas, ainsi vous pouvez explorer un texte sans perdre votre position d'édition actuelle. Cependant, par défaut, quand le curseur système bouge, le curseur de revue le suit. Ceci peut être désactivé.

7. Le Mode Navigation

Les documents complexes, tels que les pages web, sont représentés sous NVDA en utilisant le Mode Navigation. Cela concerne les documents sous Mozilla Firefox, Microsoft Internet Explorer, Google Chrome, Adobe Reader et Adobe Flash.

Le mode navigation offre une représentation platement textuelle du document dans laquelle on peut se déplacer avec les flèches de navigation. Dans ce mode, toutes les touches de commande du mode curseur fonctionneront (ex : dire tout, annonce du formatage, touches de navigation dans les tableaux etc). Des informations indiquant si le texte est un lien, un titre etc sont annoncées avec le texte pendant que vous vous déplacez.

Presser la barre d'espace ou la touche entrée passera également NVDA en mode formulaire si le champ le permet. Presser la touche échap repassera NVDA en mode navigation. 9. Configurer NVDA

9.1. Préférences

La plupart des paramètres de NVDA peuvent être modifiés en utilisant les boîte de dialogue accessibles par le sous-menu Préférences du menu NVDA. Dans toutes les boîtes de dialogue de NVDA, appuyez sur le bouton Accepter pour valider vos modifications. Pour annuler les modifications, appuyez sur le bouton Annuler ou sur la touche échap. Certains paramètres peuvent aussi être modifiés en utilisant des raccourcis clavier que vous trouverez dans les sections ci-dessous.

9.1.1. Paramètres Généraux (NVDA+control+g)

La boîte de dialogue Paramètres généraux se trouve dans le menu Préférences. Elle contient les options suivantes :

Langue

C'est une liste déroulante qui permet de choisir la langue de l'interface utilisateur et des messages de NVDA. La liste comporte de nombreuses langues et le dernier élément est dénommé "Windows". Ce choix indique à NVDA d'utiliser la langue de Windows.

Noter qu'il faut relancer NVDA quand on change la langue. NVDA vous demandera si vous désirez le redémarrer quand vous changez la sélection. Appuyez sur Accepter et NVDA redémarrera.

Sauvegarder la configuration en quittant

C'est une case à cocher qui, une fois cochée, indique à NVDA de sauvegarder automatiquement la configuration courante lorsqu'on arrête NVDA.

Avertir avant de quitter NVDA

Cette option est une case à cocher vous permettant d'indiquer si une confirmation doit être demandée quand vous voulez quitter NVDA. Quand elle est cochée, un dialogue apparaîtra quand vous voudrez quitter NVDA vous demandant si oui ou non vous voulez quitter.

Niveau de journalisation

C'est une liste déroulante qui vous permet de choisir la quantité d'information que NVDA consignera dans son journal durant son exécution. Généralement, l'utilisateur n'aura pas besoin de modifier ce paramètre dans la mesure où peu d'informations sont consignées. Cependant, si vous voulez fournir des informations lors d'un rapport d'erreur, il est peut-être bon d'augmenter cette valeur.

Démarrer NVDA automatiquement après ma connexion à Windows

Si cette option est activée, NVDA démarrera automatiquement dès que vous vous connecterez à Windows. Cette option n'existe que dans les versions installées de NVDA.

Utiliser NVDA sur l'écran de connexion à Windows (nécessite des privilèges administrateur)

Si vous vous connectez à Windows en fournissant un nom d'utilisateur et un mot de passe, l'activation de cette option permettra à NVDA de démarrer automatiquement dès l'écran de connexion au démarrage de Windows. Cette option n'existe que dans les versions installées de NVDA.

Utiliser les paramètres actuellement sauvegardées sur l'écran de connexion et les autres écrans sécurisés

L'appui sur ce bouton copie votre configuration utilisateur actuellement sauvegardée dans le répertoire système de NVDA, ainsi, NVDA peut l'utiliser quand il s'exécute sur l'écran de connexion et les autres écrans sécurisés. Pour être sûr que tous vos paramètres sont transférés, sauvegardez d'abord vos paramètres avec la commande NVDA+control+c ou par Sauvegarder la configuration dans le menu NVDA. Cette option n'existe que dans les versions installées de NVDA.

9.1.2. Choix du synthétiseur (NVDA+control+s)

Une fois que vous aurez choisi votre synthétiseur, appuyez sur Accepter et NVDA chargera le synthétiseur choisi. S'il y a une erreur au chargement du synthétiseur, NVDA vous en informera par un message puis continuera à fonctionner avec le synthétiseur précédent.

Synthétiseur

Cette option vous permet de choisir le synthétiseur de parole qui sera utilisé par NVDA.

Pour une liste des synthétiseurs de parole pris en charge par NVDA, reportez-vous à la section Synthétiseurs de parole pris en charge.

L'élément particulier "Pas de parole" apparaîtra toujours dans cette liste. Cela peut être utile pour quelqu'un qui veut utiliser NVDA seulement avec le Braille, ou peut-être pour un développeur voyant qui ne veut utiliser que la visionneuse de parole.

9.1.3. Paramètres vocaux (NVDA+control+v)

Le dialogue Paramètres vocaux qui se trouve dans le menu Préférences, contient des options vous permettant de changer le son de la voix. Pour une manière plus rapide de contrôler les paramètres vocaux depuis n'importe où, reportez-vous à la section boucle des paramètres synthétiseur.

Ce dialogue contient les options suivantes :

Une liste déroulante qui donne la liste de toutes les voix du synthétiseur courant. A l'aide des flèches de direction, on peut entendre les différents choix.

Variante

Une liste déroulante permettant de choisir la variante utilisée par le synthétiseur. Actuellement, seule eSpeak propose ce paramètre. Les variantes eSpeak sont comparables à des voix, dans la mesure où elles modifient significativement les attributs de la voix eSpeak.

Débit

Cette option vous permet de modifier le débit de la parole. C'est un potentiomètre gradué de 0 à 100, (0 étant la vitesse la plus lente et 100 la plus rapide).

Hauteur

Cette option vous permet de modifier la hauteur de la voix. C'est un potentiomètre gradué de 0 à 100, (0 pour le son le plus grave et 100 pour le plus aiguë).

Volume

Cette option vous permet de modifier le volume de la voix. C'est un potentiomètre gradué de 0 à 100, (0 étant le volume le plus bas et 100 le plus élevé).

Un potentiomètre vous permettant de choisir le niveau d'inflexion (variation en hauteur de la voix) que le synthétiseur devra utiliser.

Si le changement automatique de langue est activé, cette case à cocher vous permet de choisir si les changements de dialecte doivent être pris en compte en plus des changements de langues réels. Par exemple, un texte peut être en Anglais Américain et comporter des passages marqués comme Anglais Britannique. Si cette option est activée, le synthétiseur respectera les changements d'accent. Par défaut, cette option est désactivée.

Ce champ d'édition vous permet d'entrer la valeur du changement de hauteur de la voix à l'annonce d'une lettre majuscule. Cette valeur est un pourcentage, une valeur négative réduit la hauteur, une valeur positive l'augmente. Pour n'avoir aucun changement de hauteur, entrez la valeur 0.

Une case à cocher qui, cochée, indique à NVDA de dire maj pour une lettre en majuscule quand on l'atteint par les flèches de direction ou lors de la frappe. Normalement, NVDA énonce chaque lettre majuscule avec un ton plus aiguë, mais ceci n'est pas correctement pris en charge par certains synthétiseurs, si bien que cette option est parfois utile.

Annoncer les majuscules par des bips

Quand cette case à cocher est cochée, NVDA émettra un petit bip chaque fois qu'il rencontre un caractère majuscule. Comme l'option "Dire maj avant les majuscules" cette option est utile pour les synthétiseurs ne pouvant pas modifier la hauteur de leur voix pour les majuscules.

Utilisez la fonction d'épellation si elle est supportée

Certains mots ne comportent qu'un seul caractère, mais la prononciation est différente selon que le caractère est prononcé comme un caractèreunique (épellation) ou comme un mot. Par exemple, en Français, "y" est à la fois une lettre et un mot. Cette option permet au synthétiseur de différencier les deux cas si le synthétiseur le supporte. La plupart des synthétiseurs le supportent.

En général, cette option devrait être activée. Cependant, certains synthétiseurs utilisant les API de parole Microsoft se comportent bizarrement quand cette option est activée. Si vous rencontrez des problèmes à l'épellation de caractères, essayez de désactiver cette option.

9.1.12. Dictionnaires de prononciation

Le menu "dictionnaires de prononciation" du menu "préférences" contient des dialogues qui vous permettent de gérer la manière dont NVDA prononce certains mots ou phrases. Il y a actuellement trois types de dictionnaires. Ce sont :

  • Défaut : les règles de ce dictionnaire affectent la parole dans tout NVDA.
  • Voix : les règles de ce dictionnaire n'affectent que la voix en cours d'utilisation.
  • Temporaire : les règles de ce dictionnaire affectent tout NVDA mais seulement pour la session en cours.

Tous les dialogues des dictionnaires contiennent une liste de règles qui seront utilisées pour produire la parole. Chaque dialogue contient aussi des boutons "Ajouter", "Editer" et "Supprimer"

Pour ajouter une règle au dictionnaire, pressez le bouton "Ajout", remplissez les champs dans la boîte de dialogue puis pressez "Accepter". Vous verrez ainsi votre nouvelle règle dans la liste des règles. Cependant, pour être sûr que votre règle est vraiment sauvegardée, n'oubliez pas de presser le bouton "Accepter" pour quitter le dialogue de dictionnaire une fois que vous aurez fait toutes vos modifications.

Les règles des dictionnaires de NVDA vous permettent de remplacer une chaîne de caractères par une autre. Par exemple, vous pourriez vouloir que NVDA prononce "le mot grenouille" chaque fois qu'il rencontre le mot "oiseau". Dans le dialogue d'ajout de règles, la manière la plus simple de faire cela est d'entrer le mot "oiseau" dans la zone "modèle" et le mot "grenouille" dans la zone de remplacement. Vous pouvez aussi ajouter un commentaire décrivant la règle dans la zone commentaire, quelque chose comme "Remplacer oiseau par grenouille".

Cependant, les dictionnaires de NVDA font bien mieux que le simple remplacement de mot. Le dialogue Ajout de règles contient une case à cocher permettant de choisir si vous voulez que la règle respecte la casse pour que NVDA tienne compte des majuscules et minuscules. Par défaut, NVDA ne tient pas compte de la casse. Une autre case à cocher vous permet de définir si votre modèle est une expression régulière. Une expression régulière est un modèle contenant des symboles particuliers vous permettant de trouver une correspondance sur plus d'un caractère à la fois, ou seulement sur les lettres ou les chiffres etc. Les expressions régulières ne sont pas expliquées dans ce manuel, mais on trouve, sur le web, beaucoup de tutoriels qui vous donneront plus d'informations.

9.1.13. Prononciation des symboles et ponctuations

Ce dialogue vous permet de modifier la façon dont les ponctuations et autres symboles sont prononcés ainsi que le niveau de symbole auquel ils sont annoncés.

Pour modifier un symbole, sélectionnez-le d'abord dans la liste de symboles. Le champ Remplacement vous permet de modifier le texte qui doit être prononcé à la place de ce symbole. En utilisant le champ Niveau, vous pouvez ajuster le niveau minimum auquel ce symbole doit être annoncé.

Quand vous avez fini, pressez le bouton Accepter pour sauvegarder vos modifications, ou le bouton Annuler pour les supprimer.

9.3. Fichiers de configuration et profils utilisateur

Les versions portables de NVDA enregistrent tous les paramètres, modules d'applications personnalisés et pilotes personnalisés dans un répertoire nommé userConfig situé dans le répertoire NVDA.

Les versions installées de NVDA enregistrent tous les paramètres, modules d'applications personnalisés et pilotes personnalisés dans un répertoire spécial de NVDA situé dans votre profil utilisateur Windows. Pour accéder à votre répertoire de paramètres pour une version installée de NVDA, depuis le menu Démarrer vous pouvez aller à Tous les programmes -> NVDA -> Explorer les paramètres utilisateur.

En général, cette configuration n'a pas à être modifiée. Pour modifier cette configuration, configurez nVDA à votre convenance puis demandez à NVDA de copier la configuration sur l'écran de connexion en utilisant le bouton à cet effet dans le dialogue Paramètres généraux.

Le synthétiseur eSpeak est intégré dans NVDA et ne dépend d'aucun pilote ou programme additionnel pour être installé. Par défaut, NVDA démarre en utilisant eSpeak. SAPI 4 est un ancien standard Microsoft pour les synthétiseurs de parole logiciels. Beaucoup de synthétiseurs de parole compatibles avec ce standard peuvent être achetés ou téléchargés gratuitement auprès de différentes sociétés ou sites web. Quand vous utilisez ce synthétiseur avec NVDA, la liste des voix disponibles (accessibles via le dialogue Paramètres vocaux, ou la boucle des paramètres synthétiseur) contient toutes les voix de tous les synthétiseurs SAPI 4 installés sur votre système.

SAPI 5 est un standard Microsoft pour les synthétiseurs de parole logiciels. 13. Fonctions Avancées

13.1. Redéfinir les Assignations de Touches et les Autres Gestes de Commande

L'utilisateur peut ajouter ou réassigner des gestes de commande (telles que des touches de commande) aux scripts dans un fichier spécial situé dans le répertoire de configuration NVDA de l'utilisateur. Ce fichier s'appelle gestures.ini.

Ce fichier utilise la syntaxe standard des fichiers .ini. Le fichier peut contenir plusieurs sections et chaque section peut contenir une ou plusieurs entrées.

Chaque section contient les assignations pour un script dans un module et une classe python particulier de NVDA.

  • Le nom de la section doit être le nom du module et la classe séparés par un point (.).
  • La clé de chaque entrée est le nom du script auquel le geste de commande doit être assigné. Alternativement, on peut utiliser None pour désassigner un geste d'un script auquel il était précédemment assigné. Chaque entrée ne peut apparaître qu'une fois par section, y compris None.
  • La valeur de chaque entrée est une liste d'identifiants de gestes séparés par une virgule, pour chaque geste de commande qui doit être assigné. Les identifiants de geste terminés par une virgule doivent être entourés de quotes (" ou ').

Les identifiants de geste comprennent un code de matériel sur deux lettres (kb pour clavier, br pour terminal braille), un sous-code une disposition entre crochets, le signe deux-points, puis un ou plusieurs noms de touches séparés par le signe plus (+). Pour les gestes s'appliquant au clavier, la partie entre crochets correspond à la disposition clavier et est optionnelle. Si elle n'est pas indiquée, le geste s'appliquera à toutes les configurations clavier. Pour les gestes s'appliquant au terrminal braille, la partie entre crochet indique le terminal braille spécifique est est obligatoire.

Pour découvrir les identifiants de gestes, les noms de scripts, les noms de classes et de modules qui les contiennent, vous pouvez :

  1. Activer l'aide des commandes.
  2. Activer le geste (presser la touche).
  3. Désactiver l'aide des commandes.
  4. Activer la visionneuse de journal dans le menu outils de NVDA.
  5. Examiner les entrées récentes du journal. L'une d'entre elle devrait vous donner des informations au sujet du geste de commande que vous venez d'entrer, incluant le module, la classe et le script s'il est assigné.

Voici un exemple de la manière dont vous pourriez assigner NVDA+maj+t au script donnant la date et l'heure.

Pour trouver le nom du script, module.class pour date time, vous devez activer l'aide des commandes puis pressez NVDA+f12 qui est le geste standard pour le script date time. Désactivez ensuite l'aide des commandes puis ouvrez la visionneuse de journal.

Vers la fin du journal, vous devriez voir : 

  INFO - inputCore.InputManager._handleInputHelp (13:17:22):
  Input help: gesture kb(desktop):nvda+f12, bound to script dateTime on globalCommands.GlobalCommands

Vous pouvez en déduire que le nom du script est dateTime et que le module.class est globalCommands.globalCommands.

Si le fichier n'existe pas encore, vous devez créer un fichier nommé gestures.ini dans le répertoire de configuration utilisateur et y insérer les lignes suivantes : 

  [globalCommands.GlobalCommands]
  	dateTime = kb:NVDA+maj+t

Cela assignera la combinaison NVDA+maj+t (dans toutes les configurations de clavier) au script dateTime.

Notez que l'assignation originale NVDA+f12 fonctionnera toujours. Si vous voulez supprimer cette assignation, ajoutez la ligne suivante : 

  	None = kb:NVDA+f12

Bien que vous puissiez assigner un script à n'importe quelle touche, il peut être problématique d'utiliser la touche alt du clavier. NVDA passe les touches telles que alt, maj et ctrl au système d'exploitation même si elles doivent déclencher un script. Ainsi, si vous utilisez la touche alt dans une combinaison, il se peut qu'elle ouvre la barre de menu tout en déclenchant le script. il est donc préférable de n'utiliser que maj, ctrl ou la touche NVDA.

13.2. Personnalisation Avancée de la Prononciation des Symboles

Il est possible de personnaliser la prononciation des ponctuations et autres symboles au-delà de ce que permet le dialogue Prononciation des symboles et ponctuations. Par exemple, vous pouvez indiquer si un symbole particulier doit être envoyé au synthétiseur pour créer une pause ou modifier une inflexion et vous pouvez ajouter vos propres symboles.

Pour faire cela, vous devez éditer le fichier contenant les informations de prononciation des symboles dans votre répertoire utilisateur NVDA. Le fichier s'appelle symbols-xx.dic où xx est le code de la langue. Le format de ce fichier est documenté dans la section Prononciation des Symboles du guide de développement NVDA disponible dans la section Développement du site NVDA. Cependant, l'utilisateur ne peut pas définir de symboles complexes.
